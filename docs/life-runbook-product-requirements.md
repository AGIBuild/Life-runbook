# 人生哨兵 / LifeRunbook 产品需求设计

## 1. 产品定位

**人生哨兵 / LifeRunbook** 是面向个人、家庭和关键岗位人群的关键事件预案系统。

它不是普通提醒 App、健康打卡 App 或紧急求救 App，而是让用户提前配置失联、出行、离职、家庭应急、数字资产交接等关键场景的处理方案，并在触发条件满足时自动提醒本人、通知可信联系人、共享必要信息和保留执行记录。

核心定位：

> 关键时刻，替你启动人生预案。

## 2. 核心问题

人在关键人生节点中经常遗漏重要动作：

- 独自出行、夜跑、户外、出差时失联，家人无法及时判断风险。
- 离职、换手机、搬家、换城市时，账号、文件、资料和身份绑定没有清理。
- 家庭成员突发事故时，亲属不知道保险、证件、联系人、资产、医疗信息在哪里。
- 程序员、创业者、创作者、投资者拥有大量数字资产，家人不知道如何接管。
- 企业关键岗位人员突然失联或离职时，业务连续性受影响。

现有日历、提醒、备忘录只能被动提醒，无法形成“预案 + 触发 + 通知 + 留痕”的闭环。

## 3. 目标用户

### 3.1 个人用户

- 独居人群
- 频繁出差者
- 自驾、徒步、骑行、夜跑用户
- 程序员、创业者、自由职业者、投资者
- 有较多账号、服务器、域名、钱包、密码库、重要文件的人

### 3.2 家庭用户

- 有老人、孩子或长期异地成员的家庭
- 有保险、房产、车辆、医疗资料、重要证件的中产家庭
- 希望建立家庭应急清单和数字资产交接机制的人群

### 3.3 专业与企业用户

- 保险顾问、律师、财富顾问、养老机构
- 企业创始人、财务负责人、运维负责人、项目负责人
- 希望降低关键人风险的小型企业或创业团队

## 4. 产品原则

1. **预案优先**：产品核心不是监控，而是提前准备关键场景处理流程。
2. **本地优先**：敏感信息默认本地加密存储，云同步必须可选。
3. **用户确认优先**：高风险动作必须经过用户授权与多级确认。
4. **低误报优先**：宁可第一版少做触发器，也不要复杂误判。
5. **AI 降低配置成本**：AI 不直接接管生死决策，只辅助生成、检查和解释预案。
6. **不承诺救援**：产品只提供预案触发、通知和记录，不替代报警、医疗、法律或救援服务。

## 5. MVP 范围

第一版只做 5 个高价值预案模板。

### 5.1 失联预案

用户设置确认周期、宽限时间、紧急联系人和通知内容。

触发流程：

1. 到达确认时间后推送提醒用户确认状态。
2. 用户未响应时进行第二次提醒。
3. 继续超时后通知预设联系人。
4. 通知内容包含用户预设说明、最后已知位置、当前预案名称和建议动作。

### 5.2 独自出行预案

适用于自驾、夜跑、徒步、骑行、出差、独自赴约。

字段：

- 出发时间
- 预计结束时间
- 出发地
- 目的地
- 交通方式
- 同行情况
- 车牌/航班/酒店信息
- 联系人

触发条件：

- 到点未确认
- 超过宽限时间未确认
- 可选地理围栏到达/离开

### 5.3 数字资产预案

记录重要数字资产的“获取路径”和“处理说明”，不直接要求用户存明文密码。

资产类型：

- 密码管理器
- 邮箱
- 手机号码
- Apple ID / Google Account
- GitHub / GitLab
- 域名
- 服务器
- 云服务
- 加密钱包
- 订阅服务
- 重要文件位置

敏感字段必须分级加密，紧急联系人不能默认访问。

### 5.4 离职/换工作预案

面向离职、换公司、换设备、交接账号等场景。

清单包括：

- 公司设备归还
- 公司账号退出
- 个人资料备份
- 1Password / Bitwarden / iCloud / Google 绑定检查
- GitHub、邮箱、域名、服务器权限检查
- 社保、公积金、薪资、报销、个税事项检查

### 5.5 家庭应急预案

面向突发疾病、事故、家庭成员失联、证件丢失等场景。

资料类型：

- 家庭联系人
- 医疗信息
- 保险信息
- 车辆信息
- 房产信息
- 证件位置
- 常用医院
- 紧急联系人
- 宠物/老人/孩子照护说明

## 6. 核心功能

### 6.1 用户与联系人

- 手机号或邮箱登录
- 可信联系人管理
- 联系人确认邀请
- 联系人权限分级
- 联系人备用联系方式

权限等级：

- 仅接收通知
- 可查看应急卡片
- 可查看指定预案
- 可在紧急条件满足后查看敏感资料

### 6.2 预案管理

- 创建预案
- 编辑预案
- 启用/暂停预案
- 手动启动预案
- 手动解除预案
- 预案归档
- 定期复查提醒

### 6.3 触发器

MVP 只支持可靠触发器：

- 定时确认
- 指定时间未响应
- 指定事件结束后确认
- 手动启动
- 手动解除
- 地理围栏可作为第二阶段功能

第一版不做：

- 自动判断离职
- 自动读取企业邮箱
- 自动分析通知内容
- 自动医疗异常判断

### 6.4 响应动作

MVP 支持：

- App 推送
- 短信通知
- 邮件通知
- 最后已知位置分享
- 应急卡片展示
- 待办清单生成
- 触发日志记录

第二阶段支持：

- 电话呼叫
- 微信/企业微信通知
- Apple Shortcuts
- iCloud / Google Drive 备份触发
- 家庭成员协同确认

### 6.5 AI 预案助手

AI 用于降低配置门槛，不直接执行高风险动作。

能力：

- 自然语言生成预案
- 推荐触发条件
- 检查预案缺口
- 生成联系人通知文案
- 生成家庭说明书
- 定期建议更新预案

示例输入：

> 我下周一个人去甘南自驾，如果晚上 10 点之后 8 小时没确认，就通知我老婆和朋友，并告诉他们我的车牌、酒店、保险和最后位置。

AI 输出：

- 预案名称
- 触发时间
- 宽限时间
- 联系人
- 通知内容
- 需要补充的信息
- 风险提示

## 7. 数据与隐私

敏感数据分级：

- 普通数据：预案名称、创建时间、模板类型
- 私密数据：联系人、位置、出行计划
- 高敏数据：数字资产说明、保险、证件、账号路径

要求：

- 本地加密存储
- 云端同步默认关闭
- 云端同步必须端到端加密
- 高敏数据单独加密
- 支持数据导出
- 支持彻底删除
- 支持联系人权限撤销

## 8. 审计与留痕

必须记录：

- 预案创建时间
- 预案修改记录
- 触发器触发时间
- 用户响应时间
- 通知发送状态
- 联系人查看记录
- 权限变更记录

日志用于用户自查和责任边界，不用于广告画像。

## 9. 非目标

第一版不做：

- 医疗诊断
- AI 心理陪伴
- 保险销售
- 法律遗嘱生成
- 老人全天候监控
- 健康减肥打卡
- 大而全自动化平台
- 社区模板市场

## 10. 成功指标

MVP 阶段重点指标：

- 预案创建率大于 40%
- 紧急联系人添加率大于 30%
- 至少 15% 用户邀请家庭成员
- 5% 以上用户表达付费意愿
- 年付可接受价格不低于 199 元

更重要的定性指标：

- 用户是否愿意填写真实联系人
- 用户是否愿意录入真实预案
- 用户是否愿意让家人加入
- 用户是否认为产品解决了真实焦虑
